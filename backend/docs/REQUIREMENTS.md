# 需求说明文档

## 1. 项目背景

### 1.1 业务背景
鸣潮（Wuthering Waves）是一款开放世界动作游戏，游戏中的"声骸"系统类似于其他游戏的装备系统。玩家需要通过"调谐"来获取声骸的副词条属性，这是一个随机过程，涉及大量的概率计算和资源消耗。

### 1.2 痛点问题
1. **概率不透明**: 玩家不清楚各副词条的真实出现概率
2. **资源消耗难估算**: 无法预估获得目标副词条需要消耗多少资源
3. **数据无法追踪**: 调谐记录无法保存和分析
4. **评分标准不统一**: 不同角色对副词条的需求不同，缺乏统一的评分标准

### 1.3 解决方案
开发一个数据记录和分析系统，帮助玩家：
- 记录每次调谐的结果
- 统计副词条的真实概率
- 分析资源消耗情况
- 根据角色需求评估声骸价值

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 数据记录
- **副词条记录**: 记录每次调谐获得的副词条类型和数值
- **声骸记录**: 记录完整声骸的所有副词条信息
- **用户隔离**: 支持多用户独立记录数据
- **软删除**: 支持删除和恢复记录

#### 2.1.2 数据统计
- **概率统计**: 计算各副词条的出现概率
- **档位分布**: 统计各档位数值的分布情况
- **位置分布**: 分析副词条在不同位置的出现概率
- **间隔分析**: 统计目标副词条之间的间隔

#### 2.1.3 数据分析
- **双暴概率**: 计算获得双暴（暴击+暴击伤害）的概率
- **资源消耗**: 估算获得目标副词条需要的调谐器和经验
- **目标追踪**: 追踪距离上次获得目标副词条的间隔
- **声骸评分**: 根据角色需求评估声骸价值

#### 2.1.4 角色适配
- **共鸣者模板**: 为不同角色定义副词条权重
- **评分算法**: 根据角色模板计算声骸评分
- **多模板支持**: 支持 20+ 个角色的评分模板

### 2.2 辅助功能

#### 2.2.1 实时推送
- **WebSocket 支持**: 实时推送数据更新
- **广播机制**: 支持多客户端同步更新

#### 2.2.2 数据查询
- **分页查询**: 支持大数据量的分页展示
- **条件筛选**: 支持按用户、时间、类型等条件筛选
- **相似查询**: 查找具有相似副词条的声骸

## 4. 业务规则

### 4.1 副词条规则
1. **类型限制**: 共 13 种副词条类型
2. **档位限制**: 每种副词条有 4-8 个档位
3. **位置限制**: 每个声骸最多 5 个副词条
4. **唯一性**: 同一声骸不能有重复的副词条类型

### 4.2 编码规则
使用位运算编码副词条：
- **低 13 位**: 表示副词条类型（bit 0-12）
- **高位**: 表示数值档位
- **组合编码**: `(1 << 类型) | (1 << (13 + 档位))`

### 4.3 评分规则
```
声骸评分 = 主词条评分 + Σ(副词条评分)

副词条评分 = (副词条数值 × 权重) / 最大分数 × 50
```

### 4.4 资源计算
- **调谐器消耗**: 每个副词条消耗 10 个调谐器
- **调谐器回收**: 回收率 30%，每个副词条回收 3 个
- **经验消耗**: 根据副词条数量计算所需经验
- **经验回收**: 根据副词条数量计算回收经验
